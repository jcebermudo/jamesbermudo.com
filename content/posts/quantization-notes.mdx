Quantization is one of those techniques that sounds simple until you actually try to apply it. The idea is: instead of storing model weights as 32-bit floats, use 8-bit integers. You get ~4× memory reduction and faster inference on hardware with INT8 acceleration.

In practice, it's messier.

## The basic math

A floating point weight `w` is mapped to an integer `q` via:

```
q = round(w / scale) + zero_point
```

where `scale` and `zero_point` are computed per-tensor (or per-channel) to minimize quantization error. Reconstruction is:

```
w_approx = (q - zero_point) × scale
```

The quantization error is bounded by `scale / 2` per weight — which seems fine until you realize that error accumulates across matrix multiplications.

## Per-tensor vs. per-channel

**Per-tensor** quantization uses a single scale for all weights in a layer. Simple, but a single outlier weight forces the scale to be large, degrading precision everywhere else.

**Per-channel** quantization uses a separate scale per output channel. This handles outliers much better at the cost of slightly more complex kernels.

In practice: always use per-channel for weights. Per-tensor is fine for activations.

## What breaks

The weights themselves quantize reasonably well. Activations are harder — they depend on the input distribution, which you don't know at quantization time. This is why calibration matters.

You run a small calibration dataset through the model and collect activation statistics. The scale factors are then set based on observed ranges (or percentiles, to handle outliers).

Without calibration, you're essentially guessing. With good calibration, INT8 accuracy loss on most tasks is under 1% relative to FP32.

## Observations

Running PTQ on a ~1B parameter model:

| Config | Perplexity | Size | Latency |
|---|---|---|---|
| FP32 baseline | 12.4 | 4.0 GB | 1.0× |
| INT8 per-tensor | 13.1 | 1.1 GB | 0.72× |
| INT8 per-channel | 12.6 | 1.1 GB | 0.74× |

The per-channel gap is notable. 0.5 perplexity points for essentially free (no retraining, ~10 minutes of calibration).

## What I'd try next

- **GPTQ**: layer-wise quantization with second-order information. Better accuracy at 4-bit.
- **Activation-aware quantization (AWQ)**: identifies and protects salient weights. Significantly better at aggressive bit-widths.
- **KV-cache quantization**: the memory bottleneck during inference is increasingly the KV cache, not the weights.
